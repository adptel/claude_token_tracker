Overview

Your Claude Code token usage at a glance

Every time you send a message, Claude re-reads your entire conversation from the beginning — not just your latest message. That's why tokens compound fast in long sessions.

Input
Your message + full conversation history. Processed fresh every turn. Full price.
Cache Write
Claude saves the context so the next message doesn't re-process it. 1.25× input, once only.
Cache Read
Follow-up messages reuse saved context at 10× cheaper than input. More reads = more savings.
Output
Claude's response tokens. Most expensive per token — but usually the smallest volume.
Example: 3-message conversation
1
You: "Help me fix this bug." — Claude reads it fresh (input) and saves the conversation (cache write).
~4K tokens
2
You: "Try the other file." — Message 1 loads from cache (cache read = 10× cheaper). Only your new message is fresh (input).
~8K tokens
3
You: Follow-up — Messages 1–2 load from cache (cache read). New message is input. Claude's reply is output.
~14K tokens
Without caching, every message would be full input price. Cache reads save up to 10× on repeated context.
💡 Use /clear when switching tasks — it resets context so Claude stops re-reading history you no longer need.
