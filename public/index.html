Overview

Your Claude Code token usage at a glance

Every time you send a message, Claude doesn't just read your latest message. It re-reads your entire conversation from the beginning. That's why tokens add up fast.

EXAMPLE: A 3-MESSAGE CONVERSATION
Message 1
You send "Help me fix this bug." Claude reads your message fresh (input) and saves the conversation to cache (cache write).
~4K tokens
Message 2
You reply "Try the other file." Claude reuses message 1 from cache (cache read — 10× cheaper) and only processes your new message fresh (input). Then saves the updated conversation (cache write).
~8K tokens
Message 3
Claude reuses messages 1–2 from cache (cache read) and processes your new message fresh (input). Claude's reply is output.
~14K tokens
Without caching, every message would be processed at full price. Cache reads save you up to 10× on repeated context.
Input — Your message + conversation history, processed fresh. Full price.
Cache writes — Claude saves your conversation so it doesn't re-process everything next time. Costs 1.25× input, but only happens once.
Cache reads — Follow-up messages reuse the saved context. 10× cheaper than input. More cache reads = more savings.
Output — What Claude writes back. Most expensive per token, but usually the smallest amount.
Tip: Use /clear when switching tasks. It resets the conversation so Claude stops re-reading old context you no longer need.
